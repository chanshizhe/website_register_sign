\# website_register_sign 一个简单的网站用户注册与登录功能演示项目，基于 Node.js 和 Express 框架实现，包含用户注册、登录验证、验证码等基础功能，适合作为 Web 身份验证流程的入门示例。 ## 功能说明 - 用户注册：支持填写用户名、密码等信息完成注册（包含密码加密处理） - 用户登录：验证用户信息并跳转至主页面 - 验证码功能：注册/登录时的图形验证码验证，提升安全性 - 基础错误处理：包含表单验证错误、登录失败等场景的页面提示 ## 技术栈 - 后端：Node.js、Express 框架 - 前端：HTML、原生 JavaScript（页面交互） - 依赖库：  - `express`：Web 服务框架  - `lodash`：JavaScript 工具库（简化数据处理）  - `mime` 相关：处理 MIME 类型（用于静态资源响应）  - 内置模块：处理路由、请求解析等基础功能 ## 项目结构 ``` website_register_sign/ ├── app.js               # 主程序入口（服务启动、路由配置） ├── model/               # 数据模型目录（用户信息处理逻辑） │   └── index.js ├── views/               # 视图页面目录 │   ├── error.html       # 错误提示页面 │   ├── login.html       # 登录页面 │   ├── main.html        # 登录成功后的主页面 │   └── register.html    # 注册页面 ├── public/              # 静态资源目录 │   ├── javascripts/ │   │   ├── md5.js       # MD5 加密工具（用于密码加密） │   │   └── gVerify.js   # 图形验证码生成工具 │   └── stylesheets/     # 样式表文件（页面样式） ├── package.json         # 项目依赖配置 └── package-lock.json    # 依赖版本锁定文件 ``` ## 快速开始 ### 前提条件 - 已安装 Node.js（v10+ 推荐）和 npm ### 安装步骤 1. 克隆仓库到本地：   ```bash   git clone <仓库地址>   cd website_register_sign   ``` 2. 安装依赖：   ```bash   npm install   ``` 3. 启动服务：   ```bash   node app.js   ``` 4. 访问项目：   打开浏览器，输入 `http://localhost:3000`（默认端口，具体以代码配置为准）即可访问注册/登录页面。 ## 使用说明 1. 注册流程：   - 访问注册页面（`/register`）   - 填写用户名、密码等信息   - 输入图形验证码并提交   - 注册成功后可跳转至登录页面 2. 登录流程：   - 访问登录页面（`/login`）   - 输入注册的用户名和密码   - 输入图形验证码并提交   - 验证通过后跳转至主页面（`/main`） ## 注意事项 - 本项目为演示示例，未实现数据库持久化存储（用户数据可能仅存于内存），实际生产环境需对接数据库 - 密码加密使用 MD5 算法，实际应用中建议使用更安全的加密方式（如 bcrypt） - 验证码逻辑仅为基础实现，可根据需求扩展（如添加有效期、刷新功能等） ## 扩展建议 - 接入 MySQL/MongoDB 等数据库实现用户数据持久化 - 添加密码强度校验、邮箱验证等注册增强功能 - 实现登录状态管理（如 Session/Cookie 或 JWT） - 优化前端页面样式与交互体验 - 添加日志记录与安全防护（如防 SQL 注入、XSS 攻击）
